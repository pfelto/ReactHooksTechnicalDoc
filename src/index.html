Hooks at a Glance

Hooks are functions that let you use React state, lifecycle functions, or other React features from function components.

React provides a few built-in Hooks, but you can also create your own hooks to reuse stateful behavior between different components.

We will explore both built-in and custom Hooks below!

State Hook

The first built-in Hook we will look at is the State Hook, useState, which lets you use React state in a functional component. React will preserve this state between re-renders.

const [counter, setCounter] = useState(0);

useState returns a pair so it is advised to use array destructing to easily declare new state.

The pair returned is the current state value and a function that lets you update it which is similar to this.setState, but not identical.

useState takes only one argument, inital state, which is only used during the first render.

You can use the State Hook more than once in a single component.

Effect Hook

The Effect Hook, useEffect, adds the ability to perform side effects from a function component.

Side effects are operations that can affect other components and cannot be done during rendering like data fetching.

The most basic way of using the Effect Hook is to only pass it a function that contains imperative, possibly effectful code. This will run after every render including the first.

useEffect(() => { //your effect code });

Argueably the next easiest way to use the Effect Hook is to only have it run your effect after the first render. You achieve this by providing useEffect its second argument, a dependency array, with an empty array.

useEffect(() => { //your effect code that will run only once now },[]);

The Effect Hook is much more complex than the State Hook, so for more information on its usage and arguments read using the Effect Hook.

Rules of Hooks

Hooks are JavaScript functions, but they impose two additional rules:

  • Only call Hooks at the top level. Do not call Hooks inside loops, conditions, or nested functions.
  • Only call Hooks from React function components and custom Hooks.

Building Your Own Hooks

Building your own Hooks lets you reuse some stateful logic between components and removes the need to use higher-order components and render props.

Custom Hooks are more of a convention than a feature. If a function’s name starts with ”use” and it calls other Hooks, we say it is a custom Hook.

Other Hooks

  • useContext: const value = useContext(MyContext)
  • useReducer: const [state, dispatch] = useReducer(reducer, initialArg, init);
  • useCallback
  • useMemo: const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);
  • useRef: const refContainer = useRef(initialValue);
  • useImperativeHandle: useImperativeHandle(ref, createHandle, [deps])
  • useLayoutEffect
  • useDebugValue: useDebugValue(value)

See what each additional hook does at React's Hook API reference
